# 微信扫码登录 - 外部系统对接说明

本文档供外部系统（或 AI 开发助手）对接「微信扫码登录」服务使用。登录服务已部署在：**https://wxapp.topglobai.top/** 。

---

## 一、流程变化说明

| 原方案 | 现方案 |
|--------|--------|
| 用户扫固定二维码 → 在外部系统输入验证码 → 点击验证 → 登录成功 | 用户点击「微信登录」→ 打开嵌套窗口显示扫码页 → 用户扫码 → 登录服务回调并带上 code → 外部系统用 code 换 openid、写登录态 → 关闭嵌套窗口并继续原流程 |

外部系统在「用户已登录」之后的逻辑（用 openid 识别用户、发 token、跳转等）**保持不变**，只需把「获取 openid 的方式」从「验证码 + 调接口」改为「嵌套窗口扫码 + 回调拿到 code + 调接口换 openid」。

---

## 二、外部系统需配置的三项

对接前，在外部系统中配置以下三项（与 wechat-server 兼容，仅多一项「二维码链接」）：

| 配置项 | 说明 | 当前值（示例） |
|--------|------|----------------|
| **登录服务器地址** | 调用下方 API 时的根地址 | `https://wxapp.topglobai.top` |
| **访问凭证** | 调用 API 时请求头 `Authorization` 的值（由登录服务管理员提供） | 如：`GRcmjzfbQdp3`（以实际为准） |
| **二维码链接** | 嵌套窗口要打开的「扫码登录页」地址（每次打开会生成新二维码） | `https://wxapp.topglobai.top/` |

---

## 三、推荐前端流程（嵌套窗口 + 回调页）

### 3.1 用户点击「微信登录」时

1. 打开**嵌套窗口**（iframe 或 window.open 小窗口），地址为「二维码链接」并带上**回调地址**：
   ```text
   https://wxapp.topglobai.top/?redirect_uri=<回调地址>
   ```
   - **回调地址**：必须是**外部系统**的一个 URL，且需要 **urlencode**。
   - 示例：若回调为 `https://your-app.com/auth/wechat-callback`，则嵌套窗口地址为：
     ```text
     https://wxapp.topglobai.top/?redirect_uri=https%3A%2F%2Fyour-app.com%2Fauth%2Fwechat-callback
     ```

2. 用户在该嵌套窗口中看到二维码，用微信扫一扫并完成扫码。

3. 扫码成功后，登录服务会**在嵌套窗口内**跳转到：
   ```text
   <回调地址>?code=<一次性 code>
   ```
   即：外部系统的回调页会收到 query 参数 `code`。

### 3.2 外部系统「回调页」逻辑（后端或前端均可）

回调页（即上面配置的 `redirect_uri` 对应的页面）需要：

1. **从 URL 中取出 `code`**（query 参数）。
2. **用 code 调登录服务接口换取 openid**：
   - 请求：`GET <登录服务器地址>/api/wechat/user?code=<code>`
   - 请求头：`Authorization: <访问凭证>`
   - 成功响应示例：`{ "success": true, "message": "", "data": "<openid>" }`
   - `data` 即为该用户的 **openid**，可当作唯一标识，与之前「验证码方案」拿到的 openid 一致。
3. **写入登录态**：用 openid 在外部系统内创建/绑定会话（session、cookie、token 等），与之前流程一致。
4. **通知父窗口并关闭嵌套窗口**：
   - 若嵌套窗口是 **iframe**：回调页内执行 `window.parent.postMessage({ type: 'wechat-login-success', openid: '<openid>' }, '*')`（或约定好的 origin），父页面监听 `message` 事件，收到后关闭 iframe 并执行原有「登录成功」逻辑（如跳转、刷新、提示等）。
   - 若嵌套窗口是 **popup**（window.open）：回调页内可执行 `window.opener.postMessage(...)` 或 `window.close()`，由父页面根据约定关闭弹窗并继续流程。

### 3.3 父页面监听示例（iframe 场景）

```javascript
window.addEventListener('message', function (event) {
  if (event.data && event.data.type === 'wechat-login-success') {
    var openid = event.data.openid;
    // 关闭嵌套窗口（如隐藏或移除 iframe）
    // 按原有流程：用 openid 已写入的登录态做后续操作（跳转、刷新等）
  }
});
```

## 四、接口约定（与 wechat-server 兼容）

以下接口均需在请求头中携带：**`Authorization: <访问凭证>`**。

### 4.1 通过 code 获取 openid（核心）

- **URL**：`GET <登录服务器地址>/api/wechat/user?code=<code>`
- **请求头**：`Authorization: <访问凭证>`
- **说明**：code 由回调页 URL 的 query 传入，**一次性有效**，用后即废；超时或无效时 `success` 为 false。
- **成功响应**：`{ "message": "", "success": true, "data": "<openid>" }`
- **失败响应**：`{ "message": "无效或已过期的 code", "success": false }` 或参数错误时的类似 JSON。

### 4.2 获取 access_token（可选）

- **URL**：`GET <登录服务器地址>/api/wechat/access_token`
- **请求头**：`Authorization: <访问凭证>`
- **成功响应**：`{ "success": true, "message": "", "access_token": "...", "expiration": 7200 }`

## 五、对接检查清单（供 AI / 开发者自检）

- [ ] 配置三项：登录服务器地址、访问凭证、二维码链接（当前为 `https://wxapp.topglobai.top/`）。
- [ ] 用户点击「微信登录」时，打开嵌套窗口，地址为：`https://wxapp.topglobai.top/?redirect_uri=<外部系统回调地址>`，且 `redirect_uri` 已 urlencode。
- [ ] 外部系统提供「回调页」：该页 URL 即为上述 `redirect_uri`；扫码成功后会被跳转到 `redirect_uri?code=xxx`。
- [ ] 回调页逻辑：取 `code` → 请求 `GET <登录服务器地址>/api/wechat/user?code=<code>`，Header `Authorization: <访问凭证>` → 用返回的 `data`（openid）写登录态 → 通过 postMessage 通知父窗口并关闭嵌套窗口。
- [ ] 父窗口监听 `message`，收到登录成功事件后关闭嵌套窗口，并按原有流程处理（与之前「验证码登录成功」后的逻辑一致）。

## 六、简要总结（可作为 AI 提示词）

- **登录入口**：嵌套窗口打开 `https://wxapp.topglobai.top/?redirect_uri=<urlencode 后的回调地址>`。
- **登录结果**：扫码成功后跳转到 `回调地址?code=xxx`；外部系统用该 code 调 `GET https://wxapp.topglobai.top/api/wechat/user?code=xxx`，Header 带 `Authorization: <访问凭证>`，从响应 `data` 取得 openid，写入登录态。
- **前端收尾**：回调页用 postMessage 通知父窗口「微信登录成功」，父窗口关闭嵌套窗口并执行原有登录成功流程（与之前扫固定二维码+验证码登录成功后的行为一致）。
